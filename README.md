# php
1. Wat hebben we geleerd?
We hebben geleerd hoe je een database maakt met tabellen en relaties. Een techniek die we goed hebben leren gebruiken is het opzetten van php codes. Wat we lastig vonden was de verbinding tussen PHP en MySQL. Vooral het maken van de connectie en het correct ophalen van data ging vaak fout. Dit hebben we opgelost door hulp te vragen. Lieke aan haar oom en Selma aan haar vader. Op deze manier hebben wij toch veel geleerd en hebben wij het optimale uit het PO kunnen halen. Ik (lieke) vond het hel lastig om te hashen, hier heb ik dan zo hier en daar chat voor gebruikt omdat het niet lukte. Ik snap het nu wel. Wat bleek, ik moest heel simpel in phpmyadmin mijn tabellen eruit halen en heb die er handmatig toegevoegd door ze in te vullen in het aanmeldscherm. 
2. Hoe verliep de samenwerking?
We hebben de taken verdeeld: Selma deed vooral de database en Lieke werkte meer in PHP. De samenwerking ging goed, omdat we elkaar hielpen en duidelijk overlegden. Wat beter had gekund, is dat we eerder waren begonnen met de PHP-koppeling, want daar zat uiteindelijk het meeste werk in.
3. Welke keuzes hebben we gemaakt?
We kozen dit onderwerp omdat we wilden leren hoe websites met databases werken. Een belangrijke ontwerpkeuze was om de database eenvoudig en logisch te houden, zodat het koppelen met PHP overzichtelijk bleef.
4. Hoe kijken we terug op onze inzet?
We zijn trots dat het uiteindelijk gelukt is om PHP en SQL met elkaar te laten samenwerken. Bij een volgend project zouden we eerder beginnen met testen en onze code vanaf het begin beter documenteren
